# ASR 领导声纹识别服务：测试环境发布与接口文档

本文档用于把本地已验证的 `funasr-leader-asr:cpu` 镜像发布到阿里云镜像服务，并在测试环境拉取、启动、验证，最后提供给客户测试。

当前服务能力：

- 语音识别
- 说话人分离
- 领导声纹注册
- 会议音频中自动标记领导发言
- 返回每句话是否领导，以及匹配到的 `leader_id`

## 一、镜像说明

本次发布镜像：CPU 版。

本地镜像名：

```text
funasr-leader-asr:cpu
```

推荐阿里云镜像地址：

```text
crpi-cdfock1sf5ur5v0t.cn-shanghai.personal.cr.aliyuncs.com/chenxi_project/asr/asr-leader:20260528-v1
```

说明：

- 镜像内置 FunASR 模型，测试环境不需要额外挂载模型目录。
- 测试环境只需要挂载 `/data`，用于保存领导声纹数据。
- 默认服务端口容器内为 `8000`，测试环境建议映射为宿主机 `8002`。

```yaml

    image: crpi-cdfock1sf5ur5v0t.cn-shanghai.personal.cr.aliyuncs.com/chenxi_project/asr/asr-leader:20260528-v1
```

### 2. 注册领导声纹

接口会自动生成 `leader_id`，调用方不需要传 `id`，也不需要传姓名。

为了提高准确率，至少上传 2 个音频文件。建议每个文件 5 到 15 秒，尽量是同一领导本人清晰说话，背景噪声少。

#### 请求

```http
POST /leaders/enroll
Content-Type: multipart/form-data
```

#### 请求参数

| 参数 | 类型 | 必填 | 说明 |
|---|---|---|---|
| `files` | file[] | 是 | 领导声纹文件，至少 2 个 |

支持常见音频格式：

```text
wav, mp3, m4a, flac, aac, ogg, opus, webm
```

#### curl 示例

```bash
curl -X POST http://测试服务器IP:8002/leaders/enroll \
  -F "files=@leader_01.wav" \
  -F "files=@leader_02.wav"
```

#### 成功返回

```json
{
  "code": 0,
  "message": "success",
  "data": {
    "leader_id": "9f7abc6f-f089-41fd-b9ad-618c701b1561",
    "sample_count": 2,
    "dimensions": [192]
  }
}
```

#### 字段说明

| 字段 | 说明 |
|---|---|
| `data.leader_id` | 系统自动生成的领导声纹 ID，后续识别结果会返回这个 ID |
| `data.sample_count` | 本次注册的声纹文件数量 |
| `data.dimensions` | 声纹向量维度，正常为 192 |

#### 少于两个文件时返回

```json
{
  "detail": "at least two voiceprint files are required"
}
```

### 5. 语音识别 + 领导声纹识别

上传会议音频，返回每句话的识别文本、说话人编号、起止时间，以及是否领导和匹配到的 `leader_id`。

#### 请求

```http
POST /transcribe
Content-Type: multipart/form-data
```

#### 请求参数

| 参数 | 类型 | 必填 | 默认值 | 说明 |
|---|---|---|---|---|
| `file` | file | 是 | 无 | 会议音频 |
| `num_speakers` | integer | 否 | 空 | 说话人数。知道人数时建议填写，比如 3、4、10 |
| `identify_leaders` | boolean | 否 | true | 是否识别领导声纹 |
| `return_leader_scores` | boolean | 否 | false | 是否返回调试分数，客户正式调用建议 false |

#### curl 示例

```bash
curl -X POST http://测试服务器IP:8002/transcribe \
  -F "file=@meeting.wav" \
  -F "num_speakers=3" \
  -F "identify_leaders=true"
```

#### 返回示例

```json
{
  "code": 200,
  "message": "success",
  "data": [
    {
      "speaker": "0",
      "start_time": 0.53,
      "end_time": 1.48,
      "text": "嗯，开始了。",
      "is_leader": false,
      "leader_id": null
    },
    {
      "speaker": "2",
      "start_time": 29.92,
      "end_time": 36.73,
      "text": "那我负责前端部分，主要说页面渲染数据对接。",
      "is_leader": true,
      "leader_id": "9f7abc6f-f089-41fd-b9ad-618c701b1561"
    }
  ]
}
```

#### 返回字段说明

| 字段 | 说明 |
|---|---|
| `speaker` | 当前音频内的临时说话人编号，只在本次音频内有效 |
| `start_time` | 句子开始时间，单位秒 |
| `end_time` | 句子结束时间，单位秒 |
| `text` | 识别文本 |
| `is_leader` | 是否匹配到已注册领导 |
| `leader_id` | 匹配到的领导 ID；不是领导时为 `null` |

#### 调试分数示例

如需排查声纹识别，可临时增加：

```bash
curl -X POST http://测试服务器IP:8002/transcribe \
  -F "file=@meeting.wav" \
  -F "num_speakers=3" \
  -F "identify_leaders=true" \
  -F "return_leader_scores=true"
```

正式给客户调用时不建议打开 `return_leader_scores`。

## 九、阈值说明

当前推荐配置：

```text
LEADER_SPEAKER_THRESHOLD=0.35
LEADER_SEGMENT_THRESHOLD=0.50
LEADER_SCORE_MARGIN=0.03
LEADER_MIN_SEGMENT_SECONDS=1.0
```

说明：

| 参数 | 推荐值 | 说明 |
|---|---:|---|
| `LEADER_SPEAKER_THRESHOLD` | 0.35 | 主阈值。对同一个 speaker 的多段话聚合后做声纹判断，适合会议场景 |
| `LEADER_SEGMENT_THRESHOLD` | 0.50 | 单句话强证据阈值，防止普通人短句误判 |
| `LEADER_SCORE_MARGIN` | 0.03 | 第一名和第二名的最低分数差，避免两个领导分数接近时乱认 |
| `LEADER_MIN_SEGMENT_SECONDS` | 1.0 | 小于 1 秒的短句不参与声纹判断 |

为什么用这组值：

- 真实会议里声纹分数会受麦克风、噪声、说话时长影响，不能把主阈值设太高。
- 多段聚合比单句更稳定，所以 speaker 聚合阈值使用 `0.35`。
- 单句话容易误判，所以单句证据阈值使用更严格的 `0.50`。
- `0.03` 的分数差用于避免 A、B 两个领导都很像时强行匹配。

## 十、数据持久化

测试环境声纹数据保存在：

```text
/data/asr-leader/data/leaders.json
```

容器内路径：

```text
/data/leaders.json
```

只要 `./data:/data` 这个挂载不删除，容器重启、镜像更新后，已注册领导声纹不会丢失。
